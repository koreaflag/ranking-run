### 1.2 Strava 심층 분석

#### 랭킹 구조
Strava의 세그먼트 시스템은 업계 표준이 되었다.

**세그먼트 리더보드:**
- 특정 구간(segment)에서의 모든 기록을 시간순으로 정렬
- 필터: 성별, 연령대(45-54, 55-64, 65-69, 70-74, 75+), 체중, 팔로워, 클럽
- KOM/QOM (King/Queen of the Mountain) = 해당 구간 1위 타이틀
- **퍼센타일 랭킹**: "상위 12%에 위치" 같은 상대적 위치 표시 (유료 기능)
- ML 알고리즘으로 차량 이동, 전동 자전거 등 부정 활동 자동 탐지 (2025년 445만 건 제거)

**Local Legends:**
- 90일 롤링 기간 동안 특정 세그먼트를 가장 많이 완주한 사람이 획득
- 속도/페이스 무관 → 순수 빈도 기반 (일관성 보상)
- 러닝, 라이딩, 하이킹, 스키, 스노보드 세그먼트 지원
- 활동 공개 설정이 "Everyone"이어야 참여 가능

**강점:**
- 세그먼트가 "장소에 묶인 경쟁"을 만들어 로컬 커뮤니티를 자연스럽게 형성
- 퍼센타일 시스템으로 중위권 러너에게도 동기부여 ("1% 올리기" 목표)
- 부정행위 탐지에 ML 투자 → 리더보드 신뢰도 높음

**약점:**
- 세그먼트 생성은 웹에서만 가능 (모바일 불가)
- 기존 활동 데이터에서 추출하는 방식 → "달리면서 코스 만들기" 불가
- 핵심 경쟁 기능(세그먼트 리더보드, 퍼센타일)이 대부분 유료
- 연령 필터가 45세 이상부터만 → 젊은 러너 세분화 부재
- 세그먼트 과포화: 인기 지역은 수백 개의 중복 세그먼트 → 발견 어려움

#### 클럽 시스템
- 누구나 무료로 생성 (웹에서만)
- 공개/비공개 설정, 스포츠 타입별 분류
- 클럽 리더보드: 주간 기준 멤버 순위
- 이벤트 생성: 그룹 런/라이드 일정 잡기
- 게시판: 멤버 간 메시지, 정보 공유
- **국제적 클럽**: 위치 제한 없음 → 글로벌 클럽 가능하지만, 실질적 오프라인 연계는 어려움

#### 발견 메커니즘
- 클럽: 이름, 위치, 스포츠 타입으로 검색
- 세그먼트: 지도에서 시각적 탐색, 별표(star) 즐겨찾기
- "Nearby segments" 기능으로 현재 위치 기반 발견

### 1.3 Nike Run Club 심층 분석

#### 랭킹 구조
NRC는 "경쟁보다 자기 페이스" 철학을 따른다.

**챌린지 시스템:**
- 글로벌 챌린지: 월별/시즌별 커뮤니티 목표 (예: "이번 달 100km 달리기")
- 친구 챌린지: 1:1 또는 소그룹 경쟁
- After Dark Tour 연계 가상 레이스: 실제 대회 도시/거리에 맞춰 가상으로 참여
- 리더보드는 챌린지 내부에 한정 → 상시 글로벌 랭킹은 없음

**강점:**
- 완전 무료 → 진입 장벽 제로
- 코칭 콘텐츠(Coach Bennett)가 비경쟁적 러너를 끌어들임
- 물리적 이벤트(After Dark Tour 7개 도시)와 앱 내 가상 참여를 연결
- "Share Your Run Live" 실시간 위치/경로/페이스 공유

**약점:**
- 상시 랭킹 시스템 부재 → 경쟁 지향 러너에게 매력 부족
- 폐쇄적 데이터 생태계 (Strava/서드파티 동기화 차단)
- GPS 트래킹 안정성 문제 빈발
- 세그먼트/코스별 경쟁 개념 없음

#### 국제화 접근
- After Dark Tour: Sydney, London, Paris, Berlin, Tokyo, NYC, Mexico City
- 각 도시 레이스에 맞춘 가상 챌린지로 전 세계 러너 참여 유도
- 브랜드 파워로 자연스러운 글로벌 인지도 확보

### 1.4 Adidas Running 심층 분석

#### 랭킹 구조
- **월간/주간 리더보드**: 총 거리(km) 기반 순위
- 시간 필터: 이번 주, 지난 주, 이번 달, 지난 달
- 친구 간 비교에 초점
- 세그먼트 개념 없음 → 코스별 경쟁 불가

**강점:**
- 170M+ 유저 기반의 대규모 리더보드
- 도시별 adidas Runners 커뮤니티 (오프라인 연계)
- 서드파티 광고 제거로 깔끔한 UX

**약점:**
- 순수 "총 거리" 기반 리더보드 → 페이스/속도 경쟁 불가
- 세그먼트/코스 개념 없음 → 장소 기반 경쟁 불가
- 커뮤니티 기능이 상대적으로 얕음

### 1.5 Garmin Connect 심층 분석

#### 랭킹 구조
- **Connections 리더보드**: 친구(Connections) 간 주간 활동 순위
- **세그먼트**: Strava와 유사하지만 규모가 작음. 거리/속도/고도 기반
  - 세그먼트 유형: 힐 클라임, 스프린트, 다운힐
  - 노면 유형: 포장, 자갈, 흙, 혼합
- **뱃지/챌린지**: 월별 목표, Connect+ 구독자 전용 뱃지

**강점:**
- Strava Live Segments 연동 (Garmin 기기에서 실시간 세그먼트 경쟁)
- 하드웨어 데이터(심박, VO2max 등)와 결합된 풍부한 통계
- 세그먼트 생성이 웹에서 간편 (드래그 앤 드롭)

**약점:**
- Garmin 기기 소유자만 참여 가능 → 폐쇄적 생태계
- Connect+ 유료화로 커뮤니티 반발
- 소셜 기능이 상대적으로 약함 (피드, 댓글 등)
- 2025년 앱 리디자인 실패로 UX 신뢰도 하락

## 2. 경쟁 공백 분석 (Gap Analysis)

### 2.1 아무도 해결하지 못한 문제들

| 공백 | 설명 | RUNVS 기회 |
|------|------|-----------|
| **코스 = 유저가 직접 창작** | Strava 세그먼트는 기존 기록에서 추출, NRC/Adidas는 코스 개념 자체가 없음 | RUNVS는 달리면서 실시간으로 코스를 제작 → "크리에이터 경제" |
| **GPS 검증 기반 공정성** | Strava ML 탐지는 사후 정화, 나머지는 방어 미흡 | GPS 원시 데이터 서버 저장 + 실시간 검증 → 증명 가능한 공정성 |
| **무료 핵심 경쟁** | Strava 세그먼트 리더보드 유료화, Garmin Connect+ 유료화 | 코스 제작, 랭킹 참여, 크루 기능 모두 무료 |
| **크로스보더 코스 경쟁** | 클럽은 글로벌이지만 "같은 코스에서 국경을 넘어 경쟁"하는 개념은 부재 | 도시 대항전, 국가 대항전 → 코스를 매개로 국제 경쟁 |
| **코스 크리에이터 보상** | 세그먼트 만든 사람에게 아무 보상 없음 | 코스 제작자에게 참여자 수, 누적 주행거리, 크리에이터 뱃지 제공 |
| **난이도 + 로컬라이제이션** | 세그먼트에 난이도 표시 없음, 로컬 문화 반영 미흡 | 자동 난이도 분류 + 지역 언어 코스명 + 로컬 크루 문화 |

### 2.2 유저 불만에서 발견한 기회

1. **구독 피로**: Strava/Garmin 모두 핵심 기능 유료화로 반발 → RUNVS는 무료 경쟁을 핵심 가치로
2. **데이터 감옥**: NRC 폐쇄 생태계, Strava 구독 해지 시 PR 접근 불가 → 데이터 개방 정책
3. **세그먼트 과포화**: 인기 지역에 수백 개 중복 세그먼트 → RUNVS는 코스 품질 관리 (제작자 평판, 참여자 수 기반 노출)
4. **젊은 러너 연령 필터 부재**: Strava 45세+ 시작 → RUNVS는 20대, 30대, 40대 세분화

## 3. RUNVS 고유 전략: "Course Racing Network"

### 3.1 핵심 컨셉: "코스가 곧 경기장"

기존 앱들은 "기록 후 분석"이 중심이다. RUNVS는 **"코스를 중심으로 글로벌 경쟁이 일어나는 레이싱 네트워크"**라는 고유 포지션을 잡는다.

```
[기존 앱의 모델]
러너 → 달리기 → 기록 저장 → (선택적) 세그먼트 매칭

[RUNVS 모델]
코스 크리에이터 → 코스 제작 → 코스 공개
경쟁자 → 코스 발견 → 도전 → GPS 검증 → 랭킹 등록
크루 → 코스 점령 → 도시/국가 대항전
```

### 3.2 3개 축의 경쟁 레이어

#### Layer 1: Course Leaderboard (코스별 랭킹)
개별 코스 단위의 올타임 + 시즌별 랭킹.

- **올타임 랭킹**: 해당 코스의 역대 최고 기록
- **시즌 랭킹**: 90일 롤링 (Strava Local Legends 참조하되, 시간 기반으로 차별화)
- **필터**: 성별, 연령대(18-24, 25-29, 30-34, 35-39, 40-44, 45-49, 50+), 크루
- **퍼센타일**: "이 코스에서 상위 15%입니다" (무료 제공 → Strava 차별화)
- **GPS 검증 배지**: 서버에서 원시 GPS 데이터 검증 완료된 기록에 방패 아이콘 표시

#### Layer 2: Crew Wars (크루 대항전)
크루 단위 경쟁으로 소셜 결속력 강화.

- **코스 점령 (Course Dominion)**: 특정 코스에서 크루원들의 평균 기록이 가장 좋으면 해당 코스를 "점령"
- **도시 대항전 (City Clash)**: 같은 도시 내 크루들 간의 월간 경쟁
- **글로벌 크루 리그**: 전 세계 크루들이 동일 기간 동안 지정 코스에서 경쟁

#### Layer 3: World Circuit (월드 서킷)
국경을 넘는 코스 시리즈 경쟁.

- **시즌 서킷**: 분기별로 전 세계 5-7개 코스가 "서킷"으로 묶임
- **포인트 시스템**: 각 코스 순위에 따라 포인트 → 서킷 종합 순위
- **국가 대항**: 국가별 참여자 합산 포인트로 국가 순위

#### 크루 핵심 기능 명세

**Course Dominion (코스 점령)**
- 한 크루의 멤버 중 상위 3명의 평균 기록이 다른 크루보다 빠르면 해당 코스를 "점령"
- 점령 크루의 엠블럼이 코스 상세 페이지에 표시
- 점령 교체 시 양쪽 크루에 push 알림
- 점령 기간에 따라 크루 포인트 누적

**크루 랭킹 계산:**
```
크루 점수 = Σ (코스 점령 포인트) + (주간 활동 포인트) + (멤버 성장 포인트)

코스 점령 포인트: 점령 중인 코스 수 × 해당 코스 난이도 가중치
  - easy: ×1, normal: ×1.5, hard: ×2, expert: ×3, legend: ×5
주간 활동 포인트: 크루원 총 활동 거리(km) × 0.1
멤버 성장 포인트: 크루원 PR 달성 수 × 5
```

**크루 국제화 처리:**
- 크루 생성 시 "홈 도시" 설정 (PostGIS 좌표 저장)
- 지원 언어 다중 선택 → 크루 프로필에 언어 뱃지 표시
- 크루 이름/소개는 원본 언어 그대로 유지 (자동 번역 버튼 제공)
- City Clash는 같은 도시(반경 30km 내) 크루 자동 매칭

- 해당 코스와 동일 거리를 어디서든 달리면 "가상 참여" (시간만 비교, 고도/코스 특성 차이는 핸디캡 보정)
- 현지 참여 기록에 5% 포인트 보너스 → 실제 방문 유도

### 4.6 국제화 전략

#### 다국어 처리 원칙

| 요소 | 처리 방식 | 이유 |
|------|----------|------|
| **앱 UI** | i18n 파일 기반 번역 (ko, en, ja 우선) | 표준 접근 |
| **코스명** | 원본 언어 유지 + 자동 번역 버튼 | 코스명은 고유명사 성격 |
| **크루명** | 원본 유지 (번역 불가) | 브랜드 정체성 |
| **크루 소개** | 원본 + 번역 토글 | 이해 편의 |
| **랭킹 데이터** | 숫자/단위 로케일 변환 (km/mi, 페이스 포맷) | 지역별 단위 차이 |
| **국기 표시** | 유저 프로필 국가 기반 자동 표시 | 글로벌 경쟁 시각화 |
| **시간대** | 서버 UTC 저장, 클라이언트 로컬 표시 | 표준 접근 |

## 6. 구현 우선순위 (소규모 팀 대응)

### Phase 1: Foundation (4-6주)
**목표**: 코스별 랭킹 완성 + 기본 크루 기능

1. **Course Leaderboard 강화**
   - 올타임 + 시즌 랭킹 (백엔드 + 프론트엔드)
   - 퍼센타일 계산 (무료)
   - 필터 (성별, 연령대)
   - GPS 검증 배지 표시
   - 탑 3 포디움 UI

2. **크루 기본 기능**
   - 크루 생성/가입/탈퇴
   - 크루 홈 (멤버 목록, 주간 통계)
   - 크루 내 랭킹

**백엔드 작업:**
```
- GET /api/v1/courses/{id}/leaderboard?filter=season&gender=M&age=25-29
- GET /api/v1/courses/{id}/leaderboard/my-rank
- POST /api/v1/crews
- GET /api/v1/crews/{id}
- POST /api/v1/crews/{id}/join
- GET /api/v1/crews/{id}/leaderboard
```

### Phase 2: Competition (4-6주)
**목표**: 코스 점령 + City Clash

3. **Course Dominion (코스 점령)**
   - 크루별 코스 점수 계산 로직
   - 점령 크루 표시 (코스 상세)
   - 점령 교체 알림

4. **City Clash (도시 대항전)**
   - 월간 코스 자동 선정 로직
   - 크루 순위 계산
   - City Clash 전용 화면
   - 결과 발표 + 뱃지

**백엔드 작업:**
```
- GET /api/v1/courses/{id}/dominion (점령 크루 정보)
- GET /api/v1/city-clash?city=seoul
- GET /api/v1/city-clash/{id}/leaderboard
- POST /api/v1/city-clash/{id}/participate (크루 단위 참여)
```

### Phase 3: Global (6-8주)
**목표**: World Circuit + 국제 확장

5. **World Circuit**
   - 시즌/라운드 관리 (운영 도구)
   - 포인트 시스템
   - 가상 참여 로직
   - 국가 랭킹 집계
   - World Circuit 전용 화면

6. **국제화 확장**
   - 일본어 번역 완성
   - 단위 시스템 전환
   - 국기 표시 시스템

**백엔드 작업:**
```
- GET /api/v1/world-circuit/current
- GET /api/v1/world-circuit/{season}/rounds
- GET /api/v1/world-circuit/{season}/leaderboard?type=individual|country
- POST /api/v1/world-circuit/{round}/submit (라운드 기록 제출)
```

## 8. 경쟁 대비 RUNVS 포지셔닝 요약

```
                   경쟁 중심 ←→ 개인 성장 중심
                        │
  Strava ────────●──────┤
  (세그먼트)             │
                         │
  RUNVS ──────●──────────┤  ← 경쟁 + 크리에이터
  (코스 레이싱)          │
                         │
  Garmin ─────────●──────┤
  (기기 데이터)          │
                         │
  NRC ───────────────────●
  (코칭/자기페이스)      │
                         │
  Adidas ──────────●─────┤
  (커뮤니티/거리)        │
                   │
              소셜/팀 ←→ 솔로

RUNVS 고유 포지션: "코스 크리에이터 + GPS 검증 경쟁 + 크루 대항전"
```

### 최종 차별화 포인트

| # | 차별화 | 근거 |
|---|--------|------|
| 1 | **코스가 콘텐츠**: 유저가 직접 만든 코스가 경쟁의 무대 | Strava 세그먼트 = 기록 후 추출, RUNVS = 달리면서 창작 |
| 2 | **무료 핵심 경쟁**: 랭킹, 퍼센타일, 크루 모두 무료 | Strava/Garmin 핵심 기능 유료화 반발 흡수 |
| 3 | **GPS 검증 = 신뢰**: 원시 GPS 서버 저장 + 검증 배지 | Strava ML 사후 정화 vs RUNVS 사전 검증 |
| 4 | **크루 대항전**: 코스 점령, 도시 대항전, 월드 서킷 | 팀 기반 경쟁은 개인 경쟁보다 2.3배 높은 retention |
| 5 | **코스 크리에이터 보상**: 참여자 수, 뱃지, 포인트 | 어떤 앱도 세그먼트/코스 제작자를 보상하지 않음 |
| 6 | **국경을 넘는 코스 경쟁**: 같은 코스에서 전 세계 러너가 순위 경쟁 | Strava 클럽은 글로벌이나 "같은 코스"로 묶이진 않음 |
